Multi-Language Design Options

These wireframes compare the two shortlisted UI directions for multi-language support in Recetas. Both keep recipe language separate from UI language and reuse the existing tab-like interaction style where possible.

Option A

Header Locale Dropdown

Locale dropdown lives in the header action row for maximum visibility.
Trigger shows a small flag plus language label instead of a persistent button pair.
Recipe language still sits in Basic info as a compact segmented control.
Best fit if we want lower visual weight than a permanent toggle.
Open Option A
Option C

Dedicated Locale Strip

Locale switcher gets its own secondary-tab strip under the header.
Recipe language still lives in Basic info and import review.
Reduces header crowding but adds more vertical header height.
